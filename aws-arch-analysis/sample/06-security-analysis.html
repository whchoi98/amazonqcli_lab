🌐 네트워크 보안 분석

VPC 보안 아키텍처

보안 설계 강점:
✅ Multi-VPC 아키텍처로 워크로드 격리
✅ 프라이빗 서브넷 활용으로 내부 리소스 보호
✅ Transit Gateway를 통한 중앙 집중식 라우팅 제어
✅ NAT Gateway를 통한 안전한 아웃바운드 연결

네트워크 분할 현황:
- 관리 영역 (mgmtvpc): 개발 도구 격리
- DMZ 영역 (DMZVPC): 외부 접근 서비스 배치
- 워크로드 영역 (VPC01, VPC02): 애플리케이션 분리
- 기본 VPC: 최소 사용 (권장사항 준수)

보안 그룹 분석 (24개)

보안 그룹 분류 및 평가:

🟢 우수한 설정 (16개):
- EKS 관련 보안 그룹 (4개): 세분화된 클러스터 보안
- 데이터베이스 전용 (3개): Aurora, Redis, OpenSearch 격리
- 로드밸런서 전용 (2개): 웹 트래픽 제어
- 기타 서비스 전용 (7개): 목적별 명확한 분리

🟡 개선 필요 (8개):
- 일부 과도한 권한 (0.0.0.0/0) 확인 필요
- 보안 그룹 명명 규칙 표준화
- 미사용 규칙 정리 필요

권장 개선사항:
□ 보안 그룹 규칙 정기 감사 (월 1회)
□ 최소 권한 원칙 강화
□ 소스 IP 범위 최소화
□ 설명 및 태그 표준화
