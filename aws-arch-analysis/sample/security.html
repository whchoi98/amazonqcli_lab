🛡️ 보안 그룹 위험 분석

8개의 보안 그룹이 운영되고 있으며, 일부 그룹에서 보안 위험이 발견되었습니다.

보안 그룹 VPC 인바운드 규칙 아웃바운드 규칙 위험도
default (Default VPC) Default SSH (0.0.0.0/0) All Traffic 높음
EKS-cluster-sg DMZVPC HTTPS (제한적) All Traffic 낮음
EKS-node-sg DMZVPC 클러스터 통신 All Traffic 낮음
Web-tier-sg VPC01/VPC02 HTTP/HTTPS 제한적 중간
Management-sg mgmtvpc SSH (제한적) 제한적 낮음

🔴 보안 위험 발견

  • Default VPC에서 0.0.0.0/0 SSH 허용
  • 과도한 포트 범위 허용
  • 불필요한 인바운드 규칙
  • 즉시 검토 및 수정 필요

✅ 보안 상태 양호

  • EKS 보안 그룹 적절히 구성
  • VPC별 보안 그룹 분리
  • 서비스별 전용 보안 그룹
  • 기본 보안 정책 준수
